Service(四层代理)

发表于 2024-09-03 更新于 2024-09-04 分类于 Operations ， k8s 阅读次数： Waline： 本文字数： 1k 阅读时长 ≈ 2 分钟

在 Kubernetes 中，Service 是一种关键资源，用于将一组 Pod 暴露为网络服务。它提供了一个稳定的接口（通常是 IP 地址和端口），用于访问这些 Pod，即使它们的 IP 地址会随着时间的变化而变化。Service 的主要作用是使应用程序组件之间的通信更加可靠和可扩展。

Service 的主要特点

  1. 稳定的网络端点：
    • Service 提供一个稳定的 IP 地址（ClusterIP）和 DNS 名称，即使在后台 Pod 动态变化时，这个地址也不会改变。
  2. 负载均衡：
    • Service 会自动将流量分发到与其关联的 Pod 上，确保所有 Pod 都能均匀地处理请求。这种负载均衡是基于轮询或其他简单的算法实现的。
  3. 服务发现：
    • Service 可以通过 DNS 或环境变量的方式在集群中被其他服务发现和访问。在启用了 Kubernetes DNS 插件的集群中，每个 Service 都会被分配一个 DNS 名称。
  4. 透明的 Pod 更新：
    • 当与 Service 关联的 Pod 被更新或重新部署时，Service 会自动更新其目标，确保流量仍然能够正确地分发到新的 Pod 上。

Service 的使用场景

  • 集群内部通信：Service 常用于在 Kubernetes 集群内部不同应用程序或组件之间建立稳定的通信通道。
  • 集群外部访问：通过特定类型的 Service，如 NodePort 或 LoadBalancer，可以将集群内的服务暴露给集群外部用户或系统。
  • 跨集群通信：使用 ExternalName Service 类型，可以将集群内的服务请求转发到集群外部的 DNS 名称，从而实现跨集群或跨系统的服务调用。

Service 的类型

  • ClusterIP：仅在集群内部访问，默认类型。
  • NodePort：通过集群每个节点的 IP 地址和一个指定端口暴露服务。
  • LoadBalancer：使用云提供商的负载均衡器将服务暴露到互联网。
  • ExternalName：将服务映射到外部 DNS 名称，不在 Kubernetes 内部代理流量。
