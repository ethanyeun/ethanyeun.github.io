emptyDir

  • 临时存储：数据仅在 Pod 的生命周期内有效。Pod 终止后，数据会丢失。
  • 节点本地存储：emptyDir 卷存储在节点本地磁盘上，因此它不能跨节点共享。如果 Pod 被调度到另一个节点，新节点的 emptyDir 将是空的。
  • 数据共享：Pod 中的所有容器可以通过 emptyDir 卷共享数据。

hostPath

  • hostPath 中的 type 字段：

  • type 字段用于指定挂载的类型，可以是以下几种：

  • Directory: 挂载主机上的一个目录。如果目录不存在，则会创建一个空目录。

  • DirectoryOrCreate: 挂载主机上的一个目录。如果目录不存在，Kubernetes 会自动创建该目录。

  • File: 挂载主机上的一个文件。

  • FileOrCreate: 挂载主机上的一个文件。如果文件不存在，Kubernetes 会创建该文件。

  • Socket: 挂载主机上的一个 Unix 套接字。

  • CharDevice: 挂载主机上的一个字符设备。

  • BlockDevice: 挂载主机上的一个块设备。

hostPath缺点

使用 hostPath 类型的 Volume 可以将宿主机的目录或文件挂载到容器中。这意味着容器可以直接访问宿主机的文件系统，这可能会导致数据泄露或篡改风险。

权限问题:如果容器和宿主机上的应用程序共享同一文件系统，容器可能会拥有对这些文件的读写权限，这可能会对宿主机的数据安全性造成威胁。尤其是在多租户环境中，不同的容器可能会对宿主机上的文件造成意外的修改或删除。
容器和宿主机的隔离性:

hostPath 类型的 Volume 破坏了容器和宿主机之间的隔离性。虽然这种类型的 Volume 在某些情况下是方便的，但它也意味着容器可以直接影响宿主机的文件系统，可能会带来安全问题。

生产环境使用的PVC

hostPath 卷的用量不会被视为临时存储用量。 你需要自己监控磁盘使用情况，因为过多的 hostPath 磁盘使用量会导致节点上的磁盘压力。

NFS

  • nfs 卷能将 NFS (网络文件系统) 挂载到你的 Pod 中。 不像 emptyDir 那样会在删除 Pod 的同时也会被删除，nfs 卷的内容在删除 Pod 时会被保存，卷只是被卸载。 这意味着 nfs 卷可以被预先填充数据，并且这些数据可以在 Pod 之间共享。

PersistentVolume (持久化存储卷)

  • PersistentVolume（PV）是 Kubernetes 中用来表示存储资源的 API 对象。PV 是集群中的一块存储资源，它独立于 Pod 存在，通常由管理员创建和管理。PV 定义了存储的属性，如存储容量、访问模式和存储类型（如 NFS、EBS、Ceph 等）。

PVC是如何关联PV的

两者的 storageClassName 都是 manual。

PVC 请求的存储大小为 5Gi，符合 PV 的 5Gi 容量。

PVC 和 PV 的 accessModes 都是 ReadWriteOnce。

每个持久卷会处于以下阶段（Phase）之一

  • Available

    卷是一个空闲资源，尚未绑定到任何申领

  • Bound

    该卷已经绑定到某申领

  • Released

    所绑定的申领已被删除，但是关联存储资源尚未被集群回收

  • Failed

    卷的自动回收操作失败

PersistentVolumeClaim (持久化存储卷请求)

  • PersistentVolumeClaim（PVC）是 Kubernetes 中用来请求存储资源的 API 对象。PVC 描述了应用程序所需的存储资源的要求，如存储容量、访问模式等。PVC 是用户或应用程序发起的请求，用来声明他们需要的存储资源。

  • 持久卷常用的存储方式：NFS、iSCSI、云提供商的块存储（如 Amazon EBS、Google Persistent Disk）。

    临时卷常用的类型：emptyDir、configMap、secret。

StorageClass(自动创建pv)

  • 云环境/动态应用：如果你运行的是公有云上的集群，并且应用需求频繁变动，推荐使用动态创建 PV。云提供商的动态存储插件可以自动分配和释放资源，节省运维成本。
  • 私有云/内部存储：如果你使用的是企业内部存储系统，且有严格的存储资源管理需求，手动创建 PV 可以提供更多的控制力和可预测性。
  • 混合方式：可以选择混合策略，对于需要灵活性的开发、测试环境使用动态创建，对于生产环境中的关键应用使用手动管理的 PV。这种方式能够提供灵活性和控制力的平衡。

StorageClass中只有metadata字段，所以配置起来相对于非常好配置
